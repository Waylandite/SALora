## 🚀 主要执行脚本

### run_code_search.py
**主要训练脚本**，用于代码任务的SALora搜索。

**目的**：运行双层优化以搜索最优的逐层逐模块LoRA rank

**关键特性**：
- 支持3个任务：code2nl、code2code、nl2code
- 使用Betty框架实现双层优化
- 内层循环：更新LoRA参数
- 外层循环：更新架构参数（alphas）
- 集成光谱侵入度量
- 保存完整的rank_pattern配置

**使用方法**：
```bash
python run_code_search.py \
    --task code2nl \
    --data_dir ./data/jcsd \
    --model_name Qwen/Qwen2.5-Coder-1.5B \
    --output_dir ./output_code2nl
```

**输出文件**：
- `peft_config.json`：包含rank_pattern的完整配置
- `peft_config_rank_pattern.json`：仅rank_pattern
- `model.pt`：训练的模型权重
- `architecture.pt`：架构参数
- `results.json`：评估指标

### verify_with_peft.py
**验证脚本**，使用标准PEFT验证SALora搜索结果。

**目的**：应用发现的rank配置使用标准PEFT库并评估性能

**关键特性**：
- 从peft_config.json加载rank_pattern
- 支持rank_pattern（PEFT 0.7.0+）和降级方案（单一rank）
- 使用标准PEFT LoraConfig进行训练
- 在验证集和测试集上评估

**使用方法**：
```bash
# 使用rank_pattern（推荐）
python verify_with_peft.py \
    --salora_config ./output_code2nl/peft_config.json \
    --task code2nl \
    --data_dir ./data/jcsd

# 强制使用降级方案（单一全局rank）
python verify_with_peft.py \
    --no_rank_pattern \
    ...
```

**输出文件**：
- `verification_results.json`：测试指标和rank信息
- `sample_predictions.json`：样例预测
- `best_model/`：最佳checkpoint

### analyze_salora_results.py
**分析脚本**，用于检查发现的rank配置。

**目的**：可视化和分析rank_pattern统计信息

**特性**：
- 各模块类型统计（query、key、value等）
- 层组分析（早期、中期、晚期层）
- Rank分布可视化
- 模块重要性排序

**使用方法**：
```bash
python analyze_salora_results.py ./output_code2nl/peft_config.json
```

**输出示例**：
```
✅ 使用rank_pattern (PEFT 0.7.0+)

rank_pattern: 84个层-模块组合
  层数: 12
  每层模块数: 7

各模块类型统计:
  query   : 中位数=7, 均值=6.2, 范围=[3, 8]
  key     : 中位数=2, 均值=2.1, 范围=[0, 3]  ← 经常被剪枝
  value   : 中位数=8, 均值=7.3, 范围=[4, 8]  ← 高重要性
  ...
```

## ⚙️ 配置和数据

### config.py
**配置类**，用于所有支持的任务。

**类**：
- `SALoraConfig`：基础配置类
- `Code2NLConfig`：代码摘要配置（JCSD）
- `Code2CodeConfig`：断言生成配置（ATLAS）
- `NL2CodeConfig`：代码生成配置（conCode）

**关键参数**：
```python
# LoRA设置
lora_r: int = 8                    # 最大rank
lora_alpha: int = 16               # 缩放因子
lora_dropout: float = 0.0

# 架构搜索
lambda_spectral: float = 1e-4      # 光谱惩罚权重
lambda_l1: float = 1e-3            # L1正则化权重
spectral_keep_ratio: float = 0.1   # 保留前10%奇异向量

# 训练
learning_rate: float = 3e-4        # 模型学习率
arch_learning_rate: float = 3e-4   # 架构学习率
num_epochs: int = 10
batch_size: int = 8
```

### data_loaders.py
**数据加载和预处理**，用于代码相关任务。

**类**：
- `JCSDDataLoader`：代码摘要（代码 → 自然语言）
  - 数据集：Java代码摘要数据集
  - 格式：`{"code": "...", "summary": "..."}`

- `ATLASDataLoader`：断言生成（代码 → 代码）
  - 数据集：ATLAS断言生成
  - 格式：`{"focal_method": "...", "test_prefix": "...", "assertion": "..."}`

- `ConCodeDataLoader`：代码生成（自然语言 → 代码）
  - 数据集：conCode
  - 格式：`{"nl": "...", "code": "..."}`

- `DataCollatorForSeq2Seq`：生成任务的自定义collator

**使用方法**：
```python
data_loader = get_data_loader(
    task_type='code2nl',
    data_dir='./data/jcsd',
    tokenizer=tokenizer,
    max_source_length=512,
    max_target_length=128,
)
dataset = data_loader.load_dataset()
```

### metrics.py
**评估指标**，用于所有任务。

**函数**：

**Code2NL（摘要）**：
- `compute_bleu()`：BLEU-1/2/3/4 n-gram重叠
- `compute_meteor()`：带同义词的语义相似度
- `compute_rouge_l()`：最长公共子序列

**Code2Code（断言生成）**：
- `compute_exact_match()`：精确字符串匹配
- `compute_sam()`：语义断言匹配（标准化比较）
  - 标准化 `assertTrue(x)` → `assertEquals(true, x)`
  - 标准化 `assertFalse(x)` → `assertEquals(false, x)`

**NL2Code（代码生成）**：
- `compute_exact_match()`：精确代码匹配
- `compute_bleu()`：N-gram相似度

**使用方法**：
```python
metrics = compute_metrics_by_task(
    task_type='code2nl',
    predictions=['摘要 1', '摘要 2'],
    references=['参考 1', '参考 2']
)
# 返回：{'bleu': 0.xx, 'meteor': 0.xx, 'rouge_l': 0.xx}
```

## 🧩 核心模块

### loralib/
**SALora LoRA层实现**，带有基于alpha的rank选择。

#### loralib/__init__.py
模块初始化，导出`Linear`和工具函数。

#### loralib/layers.py
**SALora Linear层**，带有可学习的rank选择。

**关键类**：`Linear`
- 继承自`nn.Linear`
- 实现带有alpha权重rank聚合的LoRA：
  ```
  ΔW = Σ α_j (u_j v_j^T)
  ```
- 支持训练期间的动态rank分配
- 与标准PyTorch优化器兼容

**使用方法**：
```python
import loralib

# 用loralib.Linear替换nn.Linear
linear = loralib.Linear(
    in_features=768,
    out_features=768,
    r=8,  # 最大rank
    lora_alpha=16,
    lora_dropout=0.0,
)
```

#### loralib/utils.py
**LoRA层的工具函数**。

**函数**：
- `mark_only_lora_as_trainable()`：冻结除LoRA外的所有参数
- 参数计数工具
- 权重初始化辅助函数

### models/
**模型包装器**，用于将SALora注入transformer模型。

#### models/__init__.py
模块初始化，导出模型创建函数。

#### models/model_wrapper.py
**核心模型适配层**。

**关键函数**：

1. **`inject_salora_to_qwen()`**
   - 将SALora注入Qwen2.5-Coder模型
   - 替换7个模块：q_proj、k_proj、v_proj、o_proj、gate_proj、up_proj、down_proj
   - 返回模型和LoRA模块映射

2. **`create_salora_model()`**
   - 自动检测模型类型并注入SALora
   - 支持：Qwen2.5、LLaMA
   - 所有模型类型的统一接口

3. **`save_lora_config_for_peft()`**
   - **最重要的函数**：生成PEFT兼容配置
   - 创建PEFT格式的rank_pattern：
     ```
     "model.layers.0.self_attn.q_proj": 7
     "model.layers.0.self_attn.k_proj": 2
     ...
     ```
   - 处理模型类型命名约定（qwen2、llama）
   - 生成rank_pattern和降级配置
   - 输出2个文件：`peft_config.json`和`peft_config_rank_pattern.json`

**名称映射**：
```python
# SALora格式 → PEFT格式
"layer.0.query" → "model.layers.0.self_attn.q_proj"
"layer.0.key"   → "model.layers.0.self_attn.k_proj"
"layer.0.gate"  → "model.layers.0.mlp.gate_proj"
```

### search/
**架构搜索组件**。

#### search/__init__.py
模块初始化，导出搜索类。

#### search/architecture.py
**架构参数（alphas）**，用于rank选择。

**关键类**：`SALoraArchitecture`
- 管理每层l和rank组件j的架构参数α_{l,j}
- 形状：(n_layers, n_modules, r_max)
- 方法：
  - `get_alpha_weights()`：获取softmax标准化的alphas
  - `get_effective_ranks()`：计算每个模块的有效rank
  - `get_rank_summary()`：按模块类型汇总rank

**有效rank计算**：
```python
# 通过softmax的Alpha权重
α = softmax(raw_alphas)

# 有效rank：加权和或阈值法
r_eff = Σ α_j  (如果 α_j > 阈值)
```

#### search/spectral_metric.py
**光谱侵入度量**，用于维持预训练知识。

**关键类**：`SpectralIntrusionMetric`

**目的**：测量LoRA更新偏离预训练权重空间的程度

**度量**：
```
R_spec = Σ α_{l,j} · ||P_l^⊥ u_{l,j}||²

其中 P_l^⊥ = I - U₀^k (U₀^k)^T
```

**解释**：
- 将LoRA更新投影到预训练权重的正交空间
- 高分数 → LoRA正在"侵入"新方向（可能遗忘）
- 低分数 → LoRA尊重预训练知识流形

**方法**：
- `register_pretrained_weights()`：存储前k个奇异向量
- `compute_intrusion_score()`：计算光谱侵入
- `compute_intrusion_score_simplified()`：训练用的高效版本
